# Git 规范

本文件定义当前仓库的 Git 提交流程约束。任何违反本文件的操作都应视为流程违规，不得以效率为由跳过。

## 分支规则
- 禁止直接在 `master` 分支上进行编码、提交或其他变更操作
- 开始编码前必须先检查当前分支
- 若发现当前分支为 `master`，必须立即停止，不得继续修改文件
- 只有切换到正确的工作分支后，才允许继续开发

## 提交前检查
- 每次 `commit` 前必须执行编译检查
- 编译失败时禁止提交
- 只有在当前变更处于可编译状态时，才允许创建提交

## Push 规则
- 禁止自动执行 `push`
- `push` 必须由用户主动触发
- 在用户未明确要求前，只允许本地整理、检查和提交，不得上传远端
- 保留用户审查代码和提交内容的机会，不得绕过该审查窗口

## Commit Message 规则
- 提交信息必须使用以下格式：`[<变更名>] <中文简述>`
- `<变更名>` 应简洁表达本次提交的主题或模块
- `<中文简述>` 应直接说明本次提交做了什么，不写模糊描述
- 禁止使用无信息量的提交信息，例如 `update`、`fix`、`test`

## 阻断条件
- 当前分支是 `master`
- 当前变更未通过编译检查
- 未经用户明确触发即执行 `push`
- 提交信息不符合规定格式

## 执行原则
- Git 流程规范优先于个人习惯
- 发现违规条件时，应先停止操作并修正流程，再继续开发
